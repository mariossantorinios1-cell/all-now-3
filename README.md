# AllNow - Final Supabase Scaffold (Delivery)

This is the FULL AllNow scaffold prepared for Supabase + Next.js + Tailwind.
It includes placeholders and example implementations for auth, realtime chat, counters (views/clicks), ads placeholders, donations (Stripe-ready), and basic pages/components.

## Important: environment variables (Vercel or .env.local)
Set these before running or deploying:

- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- NEXT_PUBLIC_GOOGLE_ANALYTICS_ID (optional)
- NEXT_PUBLIC_ADSENSE_CLIENT (optional)
- NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY (optional)
- STRIPE_SECRET_KEY (only set in Vercel as secret for server-side)

## Supabase: Create required tables (SQL)
Run these SQL commands in Supabase SQL editor to create minimal tables used by the scaffold:

```sql
-- messages table for chat
create table if not exists public.messages (
  id bigint generated by default as identity primary key,
  user_id uuid,
  username text,
  text text,
  created_at timestamptz default now()
);

-- views table for content view counts
create table if not exists public.views (
  id bigint generated by default as identity primary key,
  content_id text,
  user_id uuid,
  created_at timestamptz default now()
);

-- clicks table for click tracking
create table if not exists public.clicks (
  id bigint generated by default as identity primary key,
  target text,
  user_id uuid,
  created_at timestamptz default now()
);
```

## How to run locally
1. `npm install`
2. create `.env.local` with NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY
3. `npm run dev` and open http://localhost:3000

## Deployment to Vercel
1. Create a new project in Vercel and point to this repo or upload the ZIP.
2. Add env vars in Project Settings (the ones listed above).
3. Build command: `npm run build`
4. Deploy.
